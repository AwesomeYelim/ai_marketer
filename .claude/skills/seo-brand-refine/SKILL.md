# /seo-brand-refine — 브랜딩 개선 워크플로우

## 설명
`/seo-run`, `/seo-church`, `/seo-restaurant`, `/seo-fashion` 실행 결과를 입력받아 브랜딩 품질을 심층 분석하고, 구체적인 개선안을 도출하여 프롬프트와 콘텐츠를 반복 개선하는 follow-up 워크플로우.

## 사용법
```
/seo-brand-refine
```
직전 `/seo-run` 결과를 기반으로 자동 분석. 또는:
```
/seo-brand-refine <개선 초점 지시>
```
예시: `/seo-brand-refine USP가 너무 추상적, 더 구체적으로`

## 워크플로우 (5단계)

### Phase 1: 브랜딩 진단 (Brand Audit)
직전 파이프라인 결과에서 브랜딩 관련 산출물을 추출하여 진단한다.

**진단 항목:**

| 항목 | 평가 기준 | 소스 |
|------|----------|------|
| 포지셔닝 명확성 | "We are X for Y who Z"가 구체적이고 차별화된가? | knowledge_guide → brand_positioning |
| USP 강도 | 한 문장으로 기억에 남는가? 경쟁사와 겹치지 않는가? | knowledge_guide → usp |
| 보이스 일관성 | 블로그·SNS·기술 마크업까지 톤이 통일되는가? | planner → brand_voice_guideline, executor 결과 |
| 메시지 전달 | 1차 메시지(약속) → 2차(차별점) → 3차(근거) 계층이 지켜지는가? | planner → section_message_map, executor 본문 |
| 경쟁 차별화 | 경쟁 브랜드 갭을 실제로 공략하고 있는가? | knowledge_guide → competitor_positions, analyst → brand_kpis |
| 스토리텔링 | 브랜드 오리진 스토리가 콘텐츠에 자연스럽게 녹아있는가? | executor → brand_storytelling |
| 구조화 데이터 | Organization JSON-LD, 브랜드 메타태그가 완전한가? | developer → organization_schema, brand_meta |

**출력:** 각 항목별 점수(1-10) + 구체적 문제점 + 개선 방향

### Phase 2: 개선 우선순위 결정 (Prioritization)
진단 결과를 기반으로 개선 우선순위를 매긴다.

**우선순위 매트릭스:**
```
            영향도 높음
               │
   Quick Win   │   Strategic
   (즉시 실행)  │   (계획 후 실행)
───────────────┼───────────────
   Low Value   │   Foundation
   (후순위)     │   (기반 작업)
               │
            영향도 낮음
  노력 적음 ←──────→ 노력 많음
```

**분류 예시:**
- Quick Win: USP 문구 다듬기, 해시태그 추가, 메타태그 브랜드명 포함
- Strategic: 브랜드 스토리 전면 리라이팅, 경쟁 포지셔닝 재설계
- Foundation: 브랜드 가이드라인 문서화, 보이스 톤 매뉴얼 작성
- Low Value: 사소한 어휘 변경, 색상 미세 조정

### Phase 3: 프롬프트 개선 (Prompt Refinement)
우선순위 높은 항목에 대해 해당 에이전트의 프롬프트를 구체적으로 개선한다.

**개선 패턴:**

| 문제 유형 | 개선 대상 | 프롬프트 조정 방법 |
|----------|----------|-----------------|
| USP 추상적 | `prompts/knowledge_guide.txt` | USP 도출 규칙에 "구체적 수치/특징 포함" 조건 추가 |
| 보이스 불일치 | `prompts/executor.txt` | 보이스 적용 규칙에 금지 표현 목록 확대 |
| 스토리 부재 | `prompts/executor.txt` | 스토리텔링 삽입 위치와 분량 구체화 |
| 경쟁사 분석 피상적 | `prompts/knowledge_guide.txt` | 경쟁 브랜드 분석 항목에 "공략 전략" 필드 추가 |
| 메시지 계층 무시 | `prompts/planner.txt` | section_message_map 필수 매핑 규칙 강화 |
| Organization 스키마 불완전 | `prompts/developer.txt` | JSON-LD 필수 필드 체크리스트 추가 |

**규칙:** 프롬프트 수정 전 반드시 사용자에게 변경 내용을 보여주고 확인을 받는다.

### Phase 4: 재실행 및 비교 (Re-run & Compare)
개선된 프롬프트로 파이프라인을 재실행하고 before/after를 비교한다.

**비교 포맷:**
```
## 브랜딩 개선 비교

### Before → After
| 항목 | Before | After | 변화 |
|------|--------|-------|------|
| 포지셔닝 명확성 | X/10 | Y/10 | +Z |
| USP 강도 | X/10 | Y/10 | +Z |
| 보이스 일관성 | X/10 | Y/10 | +Z |
| 메시지 전달 | X/10 | Y/10 | +Z |
| 경쟁 차별화 | X/10 | Y/10 | +Z |
| 스토리텔링 | X/10 | Y/10 | +Z |

### 구체적 변경 사항
1. USP: "..." → "..." (더 구체적)
2. 보이스: "..." 표현 제거, "..." 표현 추가
3. ...
```

### Phase 5: 브랜드 가이드라인 문서 생성 (Brand Guideline Output)
최종 결과를 정리하여 재사용 가능한 브랜드 가이드라인 요약을 출력한다.

**출력 형식:**
```
## 브랜드 가이드라인 요약: {브랜드명}

### 포지셔닝
- Statement: ...
- USP: ...
- 핵심 차별점: 1) ... 2) ... 3) ...

### 브랜드 성격
- 키워드: 솔직한, 실용적, 감각적, 친근한
- 지양: 과장된, 공격적, 무미건조한

### 보이스 가이드
- 톤: ...
- 스타일 규칙: ...
- 선호 용어: ...
- 금지 용어: ...
- 플랫폼별 변주:
  - 블로그: ...
  - Instagram: ...
  - Twitter: ...

### 메시지 계층
- 1차 (브랜드 약속): ...
- 2차 (차별점): 1) ... 2) ... 3) ...
- 3차 (지원 근거): ...

### 경쟁 포지셔닝
| 경쟁사 | 포지셔닝 | 우리의 차별점 |
|--------|---------|-------------|
| ... | ... | ... |

### 브랜드 해시태그
- 필수: #브랜드명 #핵심키워드
- 권장: ...
```

## 반복 개선 사이클
이 워크플로우는 반복 실행할 수 있다:
```
/seo-run → /seo-brand-refine → 프롬프트 개선 → /seo-run → /seo-brand-refine → ...
```

매 사이클에서 before/after 비교를 누적하여 브랜딩 성숙도 추적이 가능하다.

## 에러 처리
- 직전 파이프라인 결과가 없으면: `/seo-run`을 먼저 실행하도록 안내
- 브랜딩 필드가 비어있으면: 해당 에이전트의 프롬프트 업데이트 필요성 경고
- 개선 후에도 점수 향상이 없으면: 프롬프트가 아닌 구조적 문제(에이전트 간 컨텍스트 전달) 점검 권장
