# /seo-prompt — 단일 에이전트 프롬프트 심층 분석

## 설명
특정 에이전트의 프롬프트 파일과 Go 코드를 함께 읽어 정합성을 확인하고, 7개 기준으로 점수를 매긴다.

## 사용법
```
/seo-prompt <agent_name>
```
예시: `/seo-prompt knowledge_guide`

유효한 에이전트: `knowledge_guide`, `planner`, `developer`, `executor`, `trust_manager`, `analyst`

## 실행 절차

### Step 1: 관련 파일 읽기
에이전트 이름을 `$ARGUMENTS`에서 추출하고 다음 파일을 읽는다:
- `prompts/{agent_name}.txt` — 시스템 프롬프트
- `internal/agent/{agent_name}.go` — Go 구현 코드

### Step 2: 코드-프롬프트 정합성 분석
- Go 코드의 출력 구조체와 프롬프트의 출력 지시가 일치하는가?
- Go 코드에서 요구하는 필드를 프롬프트가 명시적으로 안내하는가?
- 프롬프트에서 요구하는 내용이 Go 코드의 파싱 로직과 호환되는가?
- 구조화된 출력(JSON) 스키마와 프롬프트 지시 간 갭이 없는가?

### Step 3: 7개 기준 평가

각 항목 1-10점으로 평가한다:

| 기준 | 설명 |
|------|------|
| **역할 명확성** | 에이전트의 역할과 전문 영역이 명확히 정의되었는가? |
| **출력 가이드** | 기대하는 출력 형식과 구조가 구체적으로 안내되었는가? |
| **한국어 품질** | 자연스러운 한국어 지시, 한국 시장 맥락이 반영되었는가? |
| **컨텍스트 활용** | 이전 에이전트 결과를 참조하는 방법이 명시되었는가? |
| **품질 기준** | 최소 품질 기준과 제약 조건이 명확한가? |
| **에지케이스** | 예외 상황(데이터 부족, 모호한 요청 등) 처리가 안내되었는가? |
| **코드 정합성** | Go 코드의 구조체/파싱과 프롬프트 지시가 일치하는가? |

### Step 4: 분석 리포트 출력

```
## 프롬프트 분석: {agent_name}

### 총점: X/70 (평균 X/10)

### 기준별 점수
| 기준 | 점수 | 코멘트 |
|------|------|--------|
| 역할 명확성 | X/10 | ... |
| 출력 가이드 | X/10 | ... |
| 한국어 품질 | X/10 | ... |
| 컨텍스트 활용 | X/10 | ... |
| 품질 기준 | X/10 | ... |

### 코드-프롬프트 갭
- {불일치 사항}

### 개선 제안

### 제안 프롬프트 (개선 적용 시)
```

```
```

## 주의사항
- 프롬프트 파일을 직접 수정하지 않는다 (분석만)
- 수정은 `/seo-improve`를 사용한다
