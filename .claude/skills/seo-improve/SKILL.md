# /seo-improve — 에이전트 프롬프트 최적화

## 설명
에이전트 프롬프트를 분석하고 개선안을 제시한다. 사용자 승인 후 `prompts/*.txt` 파일을 업데이트한다.

## 사용법
```
/seo-improve <agent_name 또는 "all">
```
예시:
- `/seo-improve knowledge_guide` — 단일 에이전트
- `/seo-improve all` — 전체 에이전트

## 실행 절차

### Step 1: 대상 프롬프트 읽기
`$ARGUMENTS`에서 에이전트 이름을 추출한다.
- `all`이면 6개 모두: `knowledge_guide`, `planner`, `developer`, `executor`, `trust_manager`, `analyst`
- 개별 이름이면 해당 에이전트만

각 대상에 대해:
- `prompts/{agent_name}.txt` 읽기
- `internal/agent/{agent_name}.go` 읽기 (출력 구조체 확인)

### Step 2: 개선 패턴 적용
다음 패턴들을 기준으로 개선안을 도출한다:

#### 공통 개선 패턴
1. **구조화된 출력 강화**: JSON 출력 구조를 프롬프트에 명시적으로 포함
2. **한국어 품질 향상**: 한국 검색 시장 특화 지시 추가 (Naver, Kakao)
3. **컨텍스트 참조 명시**: "이전 에이전트의 {field}를 참고하여" 형태로 구체화
4. **품질 기준 수치화**: "최소 5개" → "5-10개, 각 키워드에 검색량/난이도/의도 포함"
5. **에지케이스 처리**: 데이터 부족, 모호한 요청 시 행동 지침 추가

#### 에이전트별 개선 패턴

**knowledge_guide**:
- 키워드 분류 체계 강화 (primary/secondary/long-tail)
- 경쟁 분석 프레임워크 추가
- 한국 검색 플랫폼별 키워드 차이 반영

**planner**:
- 콘텐츠 캘린더 요소 추가
- 우선순위 매트릭스 (영향력 × 실행 난이도)
- 단계별 마일스톤 설정

**developer**:
- JSON-LD 스키마 타입별 템플릿 안내
- Core Web Vitals 체크리스트
- 한국 포털 사이트 메타태그 규격

**executor**:
- 플랫폼별 최적 글자 수/해시태그 수
- 톤앤매너 가이드라인 구체화
- CTA 패턴 라이브러리

**trust_manager**:
- E-E-A-T 점수 rubric 세분화
- 팩트체크 우선순위 기준
- 수정 요청 시 구체적 지시 형식

**analyst**:
- KPI 벤치마크 기준 명시
- ROI 계산 프레임워크
- 경쟁사 분석 템플릿

### Step 3: 개선안 제시
각 에이전트에 대해 다음을 출력한다:

```
## {agent_name} 프롬프트 개선안

### 현재 문제점
1. ...

### 변경 사항 요약
1. ...

### 개선된 프롬프트 (전문)
```
{개선된 프롬프트 전체 텍스트}
```
```

### Step 4: 사용자 승인 후 적용
- 사용자에게 "이 개선안을 적용하시겠습니까?" 확인
- 승인 시 `prompts/{agent_name}.txt` 파일 업데이트
- 적용 후 변경 사항 요약 출력

## 주의사항
- **절대로 사용자 확인 없이 프롬프트 파일을 수정하지 않는다**
- 개선안은 항상 전체 프롬프트를 보여준다 (diff가 아닌 완성본)
- 기존 프롬프트의 핵심 역할 정의는 유지한다
- 한국어 자연스러움을 최우선으로 한다
